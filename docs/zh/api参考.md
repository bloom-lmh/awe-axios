# AxiosPlus 核心 API 参考

AxiosPlus 为现代开发者提供了一套功能完备的 HTTP 客户端解决方案，融合了 AOP 切面编程、依赖注入、请求增强等特性。以下是核心 API 的专业说明，帮助开发者快速掌握框架使用方式。

## 核心功能模块

### 1. AOP 切面编程

提供面向切面的编程能力，可在方法执行的不同阶段插入自定义逻辑，实现日志记录、性能监控、异常处理等横切关注点。

| API               | 说明                                                                    |
| ----------------- | ----------------------------------------------------------------------- |
| `@Aspect`         | 标记切面类，支持通过 `order` 参数指定执行优先级（数值越小优先级越高）   |
| `@Before`         | 前置通知，在目标方法执行前触发                                          |
| `@After`          | 后置通知，无论目标方法执行成功或异常，均在执行后触发                    |
| `@Around`         | 环绕通知，可完全控制目标方法的执行过程，通过 `proceed()` 方法触发原方法 |
| `@AfterReturning` | 返回通知，在目标方法成功执行并返回结果后触发                            |
| `@AfterThrowing`  | 异常通知，在目标方法抛出异常时触发                                      |

### 2. 依赖注入与组件管理

基于 IoC 容器实现依赖注入，简化组件间的耦合关系，支持多种实例作用域。

| API          | 说明                                                                   |
| ------------ | ---------------------------------------------------------------------- |
| `@Component` | 标记类为容器管理的组件，支持自定义组件名称与模块                       |
| `@HttpApi`   | 专门用于 HTTP 接口的组件装饰器，支持配置基础 URL、请求拦截器等全局参数 |
| `@Inject`    | 字段注入注解，自动从容器中获取依赖实例并注入                           |

### 3. HTTP 请求方法

封装标准 HTTP 方法，支持通过装饰器配置请求细节，简化接口定义。

| 方法装饰器 | 对应 HTTP 方法 | 说明                         |
| ---------- | -------------- | ---------------------------- |
| `@Get`     | GET            | 用于获取资源的请求           |
| `@Post`    | POST           | 用于提交资源的请求           |
| `@Put`     | PUT            | 用于全量更新资源的请求       |
| `@Delete`  | DELETE         | 用于删除资源的请求           |
| `@Patch`   | PATCH          | 用于部分更新资源的请求       |
| `@Options` | OPTIONS        | 用于获取资源支持的 HTTP 方法 |
| `@Head`    | HEAD           | 类似 GET，但仅返回响应头     |

### 4. 请求增强与控制

提供丰富的请求增强能力，满足各类场景下的请求控制需求。

| API                       | 说明                                                       |
| ------------------------- | ---------------------------------------------------------- |
| `@Retry`                  | 请求重试配置，支持设置重试次数、延迟策略及重试条件         |
| `@Debounce`               | 防抖处理，避免短时间内重复请求，支持立即执行与延迟执行模式 |
| `@Throttle`               | 节流处理，限制请求的执行频率，确保在指定间隔内仅执行一次   |
| `@TransformRequest`       | 请求数据转换，在请求发送前对数据进行加工处理               |
| `@TransformResponse`      | 响应数据转换，在接收响应后对返回结果进行加工处理           |
| `@RefAxios` / `@AxiosRef` | 注入 Axios 实例引用，便于直接操作底层请求客户端            |

### 5. 参数绑定

简化请求参数与方法参数的映射关系，自动处理参数序列化。

| API           | 说明                                                       |
| ------------- | ---------------------------------------------------------- |
| `@QueryParam` | 将方法参数绑定到 URL 查询字符串                            |
| `@PathParam`  | 将方法参数绑定到 URL 路径变量（如 `/users/:id` 中的 `id`） |
| `@BodyParam`  | 将方法参数绑定到请求体，支持 JSON、FormData 等格式         |

### 6. Mock 服务

内置 Mock 服务支持，便于前后端分离开发时的接口模拟。

| API            | 说明                                                        |
| -------------- | ----------------------------------------------------------- |
| `@Mock`        | 为接口配置 Mock 数据，支持多种场景的模拟响应                |
| `MockAPI`      | Mock 服务管理器，提供开启/关闭 Mock、重置处理器等功能       |
| `HttpResponse` | 用于构建 Mock 响应的工具类，支持 JSON、状态码、响应头等配置 |

### 类型导出

框架导出了所有核心类型定义，包括：

- AOP 相关类型（通知类型、切面上下文等）
- IoC 容器类型（依赖配置、实例作用域等）
- HTTP 请求类型（请求配置、响应结构、拦截器类型等）
- 装饰器元数据类型

可通过导入类型定义实现类型安全的开发。

### 基础客户端

`axiosPlus` 导出了经过增强的 Axios 实例，保留原生 Axios 的所有功能，同时集成了框架的拦截器、异常处理等特性。

---

通过上述 API，开发者可快速构建高可维护性的 HTTP 客户端层，结合 AOP 实现无侵入式的功能增强，利用依赖注入简化组件管理，借助 Mock 服务提升开发效率。
